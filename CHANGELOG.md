### Fixed

- **Glory Kill Eternal (NewHUD):** **`sv_fuelhud_background`** now defaults to **`3`** so **`GloryHUD.acs`** runs the Eternal branch (fonts **`CRUCRY*`**, **`CRUCAM*`**, **`BPAMMO10–14`**, **`SCFLM*`**, **`SCICB*`**). **`Fontdefs.txt`** NewHUD paths use **`GRAPHICS/HUD/glorykills/newhud/`** for case-sensitive loads. Existing saves: **Glorykill Options → Glory HUD Background → Eternal (NewHUD)** or **`resetcvar sv_fuelhud_background`**.

- **MENUDEF / Gearbox (UZDoom 4.14.3):** **`MENUDEF.txt` does not support `#include`** (`Unknown keyword '#'`). Gearbox **`OptionMenu`** / **`OptionValue`** / **`OptionString`** blocks are inlined at the end of **`MENUDEF.txt`**; **`lumps/includes/menudef_gearbox.inc`** is kept as an editable mirror only.

- **Gearbox + `gb_blur` (UZDoom 4.14.3):** ZScript does not allow `static` member fields on the old `gb_BlurBridge` helper (`Invalid qualifiers for sGearboxMenuBlur`). Gearbox’s “menu wants postprocess blur” flag now lives on **`gb_EventHandler`** as a **`ui`** field (writable from **`renderOverlay`**); **`PB_WheelBSSBlurHandler`** reads it via **`EventHandler.Find("gb_EventHandler")`**. Removed **`zscript/gearbox/display/blur.zs`**. **`gb_ViewModel.icons`** is **`Array<TextureID>`** — **`weapon_menu` / `inventory_menu`** no longer push **`int`**; **`pbWeaponWheelAllowed`** / hide checks use **`ui`** so **`inputProcess`** / **`renderOverlay`** don’t cross play↔ui incorrectly.

- **Equipment wheel after `give all`:** Slices for **Leech**, **Quick Launcher**, and **Meat Hook** no longer require token-only inventory that the console cheat often skips. **Leech** lists if **`HasLeech`** or **`PB_DTech` ≥ 4** (one throw); **Quick Launcher** lists from **`MiniHellRocketAmmo`** only ( **`HasRevGun`** no longer required); **Meat Hook** lists if **`PB_HookToken`**, **`HookAmmo`**, or **`AlreadyHaveMeatHook`**. **`BaseWeapon.dec`** / **`FireHook()`** use the same hook-ownership checks.

- **Weapon selection wheel (bind / `+PBWeaponWheel`):** The list no longer requires **`+INVENTORY.INVBAR`** only — it includes owned weapons with a normal **weapon slot** (0–9), matching the Doom HUD. This fixes an empty wheel and console spam after **`give all`** when **`ReadyWeapon` was also unset, **`BringUpWeapon()`** is used to raise a slotted weapon before opening; the **Deselect** abort in **`WorldTick`** is skipped while the weapon-pick wheel is active.

- **Kill streak announcer (UZDoom 4.14.3):** **`PB_Killstreak_Handler`** used **`CHAN_8`** for voice lines; this engine line’s ZScript only defines sound channels through **`CHAN_7`**, which caused a compile error. Announcer **`A_StartSound` / `IsActorPlayingSound`** now use **`CHAN_7`**.

- **Meat Hook / `pb_hook`:** Grapple is no longer gated by a non-zero **`pb_hook`**; owning the hook is enough. **`pb_hook`** only selects pendulum (**`2`**) vs default detach-on-re-fire; **`GrapplingHook`** net events work regardless. `FatalityRevenant2` retargeted from missing **`FRV2`** to existing **`FREV`** (same letter sequence as `FatalityRevenant`). **`FatalityPVP`** retargeted from missing **`PVPF`** to existing **`FSP7`** (pistol zombieman FP strip); `PPOD` gib spawns unchanged.
- **Shield Saw execution pinata:** `WorldThingDied` never fills `e.Damage`, so the handler’s `dmg < 1` check always failed. Eligibility is now set in `WorldThingDamaged` (`PB_ShieldSawPinataPending` when a saw hit’s pre-hit HP ≤ 20), then confirmed on death with saw `Inflictor`.
- **Hellduke Glory Kill (`FatalityHellduke6Eld`):** `HDF0` strip has no **F** lump under **`SPRITES/MONSTERS/fatalitys/eld_eld_hellduke`** (only **A–E** and **I**); state now plays **A–E** then **I** so the sixth frame is valid.
- **Pinky Glory Kill (`FatalityDemon4Eld`):** El Diablo LSLS token **4** no longer flashes past in a few tics; sequence paces like classic **FatalityDemon** (hold / CDCDCD beat) and spawns **XDeathDemonHead** before **DyingDemon**, matching the satisfying head-rip payoff of the other pinky paths.

- **ZScript weapons + Glory Kill token (`GoFatality`):** **`Fire` / `AltFire`** paths that set **`SetPlayerProperty(0, 1, 0)`** then **`Steady`** could leave **`PROP_FROZEN`** on after **`GoFatality`** cleared (movement / weapon PSprite soft-lock). **`Steady`** now waits out **`GoFatality`** then **`SetPlayerProperty(0, 0, 0)`** before returning to ready — **PB_CSSG**, **PB_BeamKatana**, **PB_ArgentSith**, **PB_MetalSniper**, **PB_NeoHMG**, **PB_M41A**, **PB_Excavator**, **BDPBattleRifle**, **Stormcast**.
- **Shield Saw + Commander SSG / fatality cancel:** **`ThrowShieldSaw`** and **`PB_CSSG.Steady`** only cleared **`SetPlayerProperty(0,0,0)`**; **`PROP_TOTALLYFROZEN`** (ice-style freeze, ledge/camera locks, etc.) could stay on and leave you unable to move with an invisible **`Steady`** weapon layer. **`ThrowShieldSaw`** (**`BaseWeapon.dec`** / **`BaseWeapon.zc`**) and **`PB_CSSG`** **`Steady`** now also **`SetPlayerProperty(0, 0, PROP_TOTALLYFROZEN)`** when unblocking. The same dual-clear is applied across other **`GoFatality`** **`Steady`** waits and **`Fire`/`AltFire`** branches that unfreeze the player (remaining ZScript guns/katanas/Stormcast/CSSG, plus shared **`PB_Weapon`** **`Steady`** in **`BaseWeapon.dec`**, and DECORATE weapons that override **`Steady`** or fatality **`Fire`** else—including LMG/SSG/X12/Super GL/HASG/Hell Pistoler/Quad Barrel/Paingiver/black-hole/Ballista/Demon Exterminator/Carbine/Riot Shield/Nailgun/Minigun/Marauder SSG/XM‑21/Lever Action **`Fire`** else, plasma/railgun/freezer/BFG Mk IV/BFG beam/Mastermind chaingun/Demon Tech/flamer/pistol, and fists **`MELEE`** including combo kick end).
- **Shield Saw + experimental executions (`pb_experimental`):** **`PB_SetPlayerExecutionProperties()`** applies **`PROP_INVULNERABILITY`**, **`PROP_FLY`**, and **`PROP_TOTALLYFROZEN`** before **`Execution_*`** states; **`ThrowShieldSaw`** previously only cleared classic freeze / totally frozen bits, so cancelling with the saw could leave you floating, unable to take damage, and with no usable weapon view. **`ThrowShieldSaw`** now calls **`PB_UnsetPlayerExecutionProperties()`** when the throw starts and again after the throw sequence (**`BaseWeapon.zc`** / **`BaseWeapon.dec`**).
- **First-person fatalities (`NEWPLAYE.dec` / `PLAYER.dec`) + shared weapon unblock:** Every fatality / spawn-recovery path that ends with **`SetPlayerProperty(0,0,0)`** (`PROP_FROZEN`) now also calls **`SetPlayerProperty(0,0,PROP_TOTALLYFROZEN)`** on the next line (same sprite prefix), so total freeze from ice, ledge, or weapon layers cannot persist after **`FatalityChecker`** finishers (e.g. Spider Mastermind / boss pool). **`BaseWeapon.dec`** / **`BaseWeapon.zc`** also pair clears on **`ThrowShieldSaw`**, **`DoKick`** end, and **`PB_WeapTokenSwitch`**. Remaining orphans on **`LMG`**, **`MINIGUN`**, **`Nailgun`**, and **`OldHMG`** deselect branches were aligned the same way.
- **Gore / footprints:** Stopped **NashGoreBloodPlane** spawn logic from forcing **0.9–1.0** actor scale on **`GrowingBloodPool`** (so **`RedBloodFootPrint*`** / smear chain keep their small **DECORATE** scales instead of ballooning ~10×). Removed erroneous **`MODELS/Definitions/GreenBlood.txt`** footprint model override that used **90³** MD3 scale (huge voxels + footprint spam could exhaust VRAM/system memory and hard-crash). Corrected **NashGore** footprint hooks: **`NashGoreFootprint`** class check in **PostBeginPlay**, **`footprintFootSide`** typo in **NashGoreActor** (left/right alternation).
- **ZScript quick melee / Shield Saw + Stormcast:** Glory-stagger melee (`FinisherToken` + line of fire) now runs the same **PermormGloryKill** chain as DECORATE **`PB_Weapon`** (new states on **`PB_WeaponBase`**), **close-range punch** is preferred **before** **`PB_Execute()`**, and throwing the **Shield Saw** clears stray **`GoFatality`** + **`SetPlayerProperty(0,0,0)`** so PSprites / weapon switching do not stay locked after a bad execution vs throw path.
